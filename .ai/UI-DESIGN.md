# Watson UI/UX 设计（改进版）

## 设计原则

1. **极简优先** - 只显示必要信息
2. **状态清晰** - 一眼看懂当前状态
3. **减少干扰** - 不要过度动画
4. **键盘友好** - 支持快捷键

## 功能设计

### 1. 状态指示（改进 Paw 的 Menubar）

**Paw 的问题：**
- 右上角爪印 + 文字占空间
- 状态文字可能很长

**Watson 改进：**
- 左下角小圆点指示状态
- 颜色编码：灰色（空闲）/ 蓝色（思考）/ 绿色（完成）/ 红色（错误）
- Hover 显示详细状态
- 不占用顶部空间

### 2. 消息状态（改进 Per-Card 状态行）

**Paw 的问题：**
- 每条消息都有状态行，重复
- 完成后状态行保留，视觉噪音

**Watson 改进：**
- 只在 streaming 时显示状态
- 完成后只显示 ✓，不显示文字
- 错误时才显示详细信息
- 状态融入消息卡片，不单独一行

### 3. 工具调用显示（改进工具步骤 UX）

**Paw 的问题：**
- 展开/折叠需要点击
- 工具步骤占用大量空间

**Watson 改进：**
- 工具调用显示为小标签（如 "🔧 file_read"）
- 默认折叠，Hover 显示详情
- 多个工具调用合并显示（如 "🔧 3 tools"）
- 不打断阅读流
