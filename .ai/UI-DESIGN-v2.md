# Watson UI/UX 设计（v2 - 基于 kenefe 反馈）

## 核心原则

1. **清爽明亮** - UI 要清晰，不能感觉模糊
2. **主动智能** - 本地助手，要做得很好用
3. **屏幕感知优先** - 当前窗口/文章 > 剪贴板
4. **状态透明** - 随时知道 Watson 在做什么
5. **人话解释** - 工具调用要翻译成人话

## 功能设计

### 1. 状态指示（改进）

**当前实现：**
- 左下角小圆点

**需要改进：**
- 鼠标悬停显示详细状态
- 显示未读消息数
- 不用打开就能知道状态

### 2. 工具使用显示（重要！）

**问题：**
- 不能只显示 "tool_use: file_read"
- 工具调用是给机器看的，不是给人看的

**解决方案：**
- 翻译成人话：
  - `file_read` → "正在读取 config.json"
  - `shell_exec` → "正在运行命令..."
  - `search` → "正在搜索..."
- 实时显示，不只是完成后

### 3. 屏幕感知

**优先级：**
1. 当前窗口内容
2. 正在看的文章
3. 剪贴板（最低优先级，涉及隐私）

**实现：**
- 集成 agent-control 的屏幕感知
- 自动获取当前窗口内容
- 不强调剪贴板

### 4. UI 清爽度

**改进点：**
- 更清晰的字体
- 更好的对比度
- 减少视觉噪音
- 流畅的动画
